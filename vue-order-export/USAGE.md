# 订单列表导出功能使用说明

## 功能概述

本系统实现了基于Vue2 + Element-UI的订单列表管理和导出功能，具备以下核心特性：

### ✨ 主要功能
1. **订单列表查询**：支持多条件筛选查询
2. **智能导出**：根据数据量自动选择导出方式
3. **分批导出**：大数据量自动分批处理
4. **进度监控**：实时显示导出进度
5. **文件压缩**：超过5000条自动压缩为ZIP

## 使用指南

### 1. 启动项目

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run serve
```

访问 http://localhost:8080

### 2. 订单查询

在页面顶部的搜索表单中，您可以通过以下条件筛选订单：

- **订单号**：支持模糊搜索
- **客户姓名**：支持模糊搜索  
- **订单状态**：下拉选择（待付款、已付款、已发货、已完成、已取消）
- **创建时间**：日期范围选择

点击"查询"按钮执行搜索，"重置"按钮清空所有条件。

### 3. 导出操作

#### 3.1 导出所有查询结果

点击"导出订单"按钮，系统会：

1. **数据量 ≤ 5000条**：直接导出为Excel文件
2. **数据量 > 5000条**：分批导出并压缩为ZIP文件

#### 3.2 批量导出选中订单

1. 勾选表格左侧的复选框选择订单
2. 点击"批量导出选中"按钮
3. 系统将选中的订单导出为Excel文件

#### 3.3 导出单个订单

点击订单行右侧的"导出"按钮，导出该订单的Excel文件。

### 4. 导出进度监控

当导出大量数据时，系统会显示进度对话框，包含：

- **总数据量**：需要导出的记录总数
- **当前进度**：已处理的记录数
- **进度百分比**：可视化进度条
- **状态信息**：当前处理状态
- **取消按钮**：可中途取消导出

## 导出文件说明

### Excel文件结构

每个Excel文件包含以下字段：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| 序号 | 记录序号 | 1 |
| 订单号 | 唯一订单编号 | ORD00000001 |
| 客户姓名 | 下单客户姓名 | 张三 |
| 客户电话 | 客户联系电话 | 13800138000 |
| 商品名称 | 订购商品名称 | 美白面膜 |
| 数量 | 购买数量 | 2 |
| 单价 | 商品单价 | 99.00 |
| 总金额 | 订单总金额 | 198.00 |
| 订单状态 | 当前状态 | 已付款 |
| 创建时间 | 订单创建时间 | 2024-01-01 10:30:00 |
| 更新时间 | 最后更新时间 | 2024-01-01 10:30:00 |

### ZIP文件结构

当数据量超过5000条时，系统会生成ZIP压缩包：

```
订单列表导出_2024-01-01.zip
├── README.txt                      # 导出说明文件
├── 订单列表_第1批_1-1000.xlsx        # 第1批数据
├── 订单列表_第2批_1001-2000.xlsx     # 第2批数据
├── 订单列表_第3批_2001-3000.xlsx     # 第3批数据
└── ...
```

**README.txt** 包含：
- 导出时间和统计信息
- 文件数量和记录分布
- 使用说明和注意事项

## 常见问题

### Q1: 导出速度慢怎么办？
**A**: 大数据量导出需要时间，系统已优化：
- 分批处理减少内存占用
- 异步操作避免界面卡顿
- 进度显示让用户了解状态

### Q2: 导出失败怎么办？
**A**: 检查以下方面：
- 网络连接是否正常
- 浏览器是否支持文件下载
- 数据是否存在异常
- 重试导出操作

### Q3: 文件打不开怎么办？
**A**: 确保：
- 使用支持的Office软件（Excel、WPS等）
- ZIP文件完整下载
- 解压后再打开Excel文件

### Q4: 如何自定义导出字段？
**A**: 修改 `src/utils/export.js` 中的字段映射：
```javascript
const excelData = orders.map((order, index) => ({
  '序号': index + 1,
  '订单号': order.orderNo,
  // 添加新字段
  '新字段': order.newField,
  // ...
}))
```

### Q5: 如何修改分批大小？
**A**: 在导出函数中修改 `batchSize` 参数，建议值：
- 内存充足：2000-5000
- 内存有限：500-1000
- 网络较慢：100-500
